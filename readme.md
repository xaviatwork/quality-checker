## Quality Score (Qscore)

The *quality score* is a measure of the *mean* quality of the `*.sh` files in the repository.

Any *finding* that ShellCheck displays is an opportunity to improve the code; the best possible result is having no *findings*.

So, no findings, means a perfect *quality score* of **0**.

Each severity level of a *finding* has a relative weight:

| Level    | Weight  |
| -----    | ------- |
| Style    | 1 (2^0) |
| Info     | 2 (2^1) |
| Warning  | 4 (2^2) |
| Error    | 8 (2^3) |

So, for a repository with 2 *Info* and 2 *Warning* findings, the resulting `qscore` would be:

```shell
            Style  Info Warning Error
qscore = 0 - 0*1 - 2*1 - 2*2  -  0*3 = -6
```

### `--format csv`

`--format csv` outputs the results in CSV format.

> `qcheck.sh` uses `;` as CSV delimiter by default (as in MS Excel files).

```shell
$ ./qcheck.sh --repo $REPO_NAME --format csv 
Repository ; style ; Info ; Warning ; Error ; Total ; Qscore
</path/to/repository> ; 0 ; 9 ; 9 ; 0 ; 18 ;  -54
```

### `--format qscore`

Outputs the *quality score* (*qscore*) of the repository.

```shell
$ ./qcheck.sh --repo $REPO_NAME --format qscore
-54
```
